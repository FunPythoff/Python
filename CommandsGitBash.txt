1. git init - tworzenie nowego repozytorium Git w bieżącym katalogu.

2. git clone - klonowanie istniejącego repozytorium Git na lokalny komputer.

3. git add - dodawanie plików do indeksu Git, przygotowując je do kolejnego commita.

4. git commit - tworzenie nowego commita, zatwierdzając zmiany w plikach w indeksie.

5. git status - wyświetlanie informacji o bieżącym stanie repozytorium, takich jak pliki dodane do indeksu lub niezatwierdzone zmiany.

6. git diff - wyświetlanie różnic między aktualnym stanem pliku a jego wersją w indeksie lub w ostatnim commicie.

7. git log - wyświetlanie historii commitów w repozytorium.

8. git branch - zarządzanie gałęziami Git, takimi jak utworzenie nowej gałęzi lub przełączanie się między istniejącymi gałęziami.

9. git checkout - przełączanie się między gałęziami lub do innych commitów.
9a. git checkout -b"nazwa_branch" - dodawanie nowego branch do repo
10. git merge - scalanie dwóch lub więcej gałęzi w jedną.

11. git pull - pobieranie najnowszych zmian z zdalnego repozytorium i scalenie ich z lokalnym repozytorium.

12. git push - wysyłanie zmian z lokalnego repozytorium do zdalnego repozytorium.
